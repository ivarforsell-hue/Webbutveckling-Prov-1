📁 Filer & Mappar

Namnge alltid filer/mappar med lowercase, utan mellanslag. Använd bindestreck istället.

  • Startsidan ska heta index.html – webbservern hittar den automatiskt.
  • Stilmall: style.css  |  Skript: script.js
  • Bilder i en mapp t.ex. images/
projekt/
├── index.html
├── style.css
├── script.js
└── images/
    └── bild.jpg

🏗️ HTML-grundstruktur

Varje sida måste ha <!DOCTYPE html>, <html>, <head> och <body>.

<!DOCTYPE html>
<html lang="sv">

<head>
  <title>Titel</title>
  <link rel="stylesheet" href="style.css">
  <script src="script.js" defer></script>
</head>

<body>
  <!-- Innehåll här -->
</body>

</html>
💡 defer = skriptet körs efter att hela HTML:en laddats klart.

🔤 Syntax – Normal & Void element

Normalt element

<p class="intro">Text</p>
  • Starttagg: <p class="intro">
  • Attribut: class = "intro"
  • Innehåll: Text
  • Sluttagg: </p>

Void element (tomt)

<img src="bild.jpg" alt="text">
<input type="text" required>
  • Ingen sluttagg!
  • Booleskt attribut: required – bara närvaron räcker

🧱 Vanliga HTML-element

Klicka på ett element för att se exempel.

h1 h2 h3Rubriker – h1 störst▾

Används för rubriker. h1 är störst och viktigast, h3 minst.

<h1>Stor rubrik</h1>
      <h2>Mellanrubrik</h2>
      <h3>Liten rubrik</h3>
pStycke▾

Används för vanliga textstycken. Webbläsaren lägger automatiskt till luft ovan och under.

<p>Det här är ett stycke text.</p>
      <p>Det här är ett annat stycke.</p>
aLänk – attribut: href▾

href anger vart länken leder. Använd target="_blank" för att öppna i ny flik.

<a href="https://google.se">Gå till Google</a>
      
      <a href="https://google.se" target="_blank">
        Öppna i ny flik
      </a>
ul ol liOordnad / ordnad lista▾

ul ger punktlista, ol ger numrerad lista. Varje punkt är ett li.

<ul>
        <li>Äpple</li>
        <li>Banan</li>
      </ul>
      
      <ol>
        <li>Första steget</li>
        <li>Andra steget</li>
      </ol>
table tr th tdTabell / rad / rubrikcell / datacell▾

table är behållaren. tr = rad. th = rubrikcell (fet). td = datacell.

<table>
        <tr>
          <th>Namn</th>
          <th>Ålder</th>
        </tr>
        <tr>
          <td>Anna</td>
          <td>17</td>
        </tr>
      </table>
imgBild (void) – src, alt▾

Void element – ingen sluttagg! src = sökväg till bilden. alt = beskrivning (viktigt för tillgänglighet).

<img src="images/katt.jpg" alt="En söt katt">
      
      <!-- Från webben: -->
      <img src="https://example.com/bild.png" alt="Beskrivning">
divBlock-behållare▾

Generisk block-behållare utan semantisk betydelse. Tar hela bredden. Används för att gruppera och styla element.

<div class="kort">
        <h2>Rubrik</h2>
        <p>Text inuti en div.</p>
      </div>
spanInline-behållare▾

Generisk inline-behållare. Stannar i texten. Används för att styla delar av text utan att bryta radflödet.

<p>Det här är
        <span style="color:red">röd text</span>
        mitt i ett stycke.
      </p>

🔮 Pseudoklasser & Pseudoelement

Pseudoklasser :

:nth-child(n)N:te barnet▾

Väljer det n:te barnet oavsett elementtyp. Kan använda siffror, odd, even eller formler som 2n+1.

li:nth-child(2) { color: red; }
      /* Väljer den andra <li> */
      
      li:nth-child(odd) { background: #eee; }
      /* Väljer 1:a, 3:e, 5:e... */
      
      li:nth-child(2n) { background: #ccc; }
      /* Väljer varannan (2, 4, 6...) */
:nth-last-child(n)N:te barnet bakifrån▾

Som :nth-child men räknar bakifrån. (1) = sista barnet.

li:nth-last-child(1) { font-weight: bold; }
      /* Väljer sista <li> */
      
      li:nth-last-child(2) { color: blue; }
      /* Väljer näst sista */
:nth-of-type(n)N:te av sin typ▾

Väljer det n:te elementet av en specifik typ – ignorerar syskon av andra typer.

p:nth-of-type(odd) { background: #f0f0f0; }
      /* Varannan <p>, börjar på 1:a */
      
      p:nth-of-type(2) { color: green; }
      /* Bara den andra <p> */
:nth-last-of-type(n)N:te av sin typ bakifrån▾

Som :nth-of-type men räknar bakifrån.

p:nth-last-of-type(1) { font-style: italic; }
      /* Sista <p> */
      
      p:nth-last-of-type(2) { color: purple; }
      /* Näst sista <p> */
:not(selector)Inte dessa▾

Väljer alla element som inte matchar selektorn inuti. Mycket användbart för undantag.

p:not(.intro) { color: gray; }
      /* Alla <p> UTOM de med klassen .intro */
      
      li:not(:last-child) { border-bottom: 1px solid #ccc; }
      /* Alla <li> utom den sista */
:is()Är någon av dessa▾

Matchar vilket element som helst i listan. Förkortar långa selektorer.

:is(h1, h2, h3) { color: navy; }
      /* Istället för: h1, h2, h3 { } */
      
      :is(ul, ol) li { margin: 0.5rem 0; }
      /* Alla <li> i både <ul> och <ol> */
:has()Har detta barn▾

Väljer ett element om det innehåller ett visst element. Kallas ibland "föräldra-selektor".

div:has(img) { border: 2px solid gold; }
      /* Alla <div> som innehåller en <img> */
      
      p:has(+ ul) { margin-bottom: 0; }
      /* <p> som följs direkt av en <ul> */

Pseudoelement ::

::first-letterFörsta tecknet▾

Stilsätter bara det allra första tecknet i ett element. Klassiskt för dekorativa dropp-caps.

p::first-letter {
        font-size: 2.5rem;
        font-weight: bold;
        float: left;
        line-height: 1;
        margin-right: 4px;
        color: darkred;
      }
::beforeLägg till innan▾

Lägger till genererat innehåll precis innan elementets innehåll. Kräver alltid content-egenskapen.

.kort::before {
        content: "→ ";
        color: gold;
      }
      
      /* Med ikon */
      .varning::before {
        content: "⚠️ ";
      }
      
      /* Tom – men används för styling */
      .linje::before {
        content: "";
        display: block;
        height: 2px;
        background: red;
      }
::afterLägg till efter▾

Lägger till genererat innehåll precis efter elementets innehåll. Kräver content.

.kort::after {
        content: " ←";
        color: gold;
      }
      
      /* Visa länkens URL efter texten */
      a::after {
        content: " (" attr(href) ")";
        font-size: 0.8em;
        color: gray;
      }
      
      /* Clearfix-trick */
      .container::after {
        content: "";
        display: block;
        clear: both;
      }

📦 Block vs Inline

Block

Tar hela bredden, börjar på ny rad.

Exempel: div p h1 ul table

Inline

Tar bara sitt eget utrymme, stannar i flödet.

Exempel: span a strong img
